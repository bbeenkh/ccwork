# PRD: 태그 기능 (Tag Feature)

**문서 버전:** 1.0
**작성일:** 2026-05-20
**상태:** 구현 완료 (v1)
**관련 컴포넌트:** `TagInput.tsx`, `NoteEditor.tsx`, `NoteItem.tsx`

---

## 1. 개요

### 배경

노트 앱에서 내용 기반 검색만으로는 노트를 빠르게 분류하고 찾기 어렵다. 사용자가 노트에 짧은 키워드 레이블(태그)을 붙여 주제별로 그룹화할 수 있으면 탐색 효율이 높아진다.

### 목표

- 노트 작성·편집 화면에서 태그를 즉시 추가·수정·삭제할 수 있어야 한다.
- 태그는 노트 목록 카드에서도 시각적으로 표시되어 한눈에 분류를 파악할 수 있어야 한다.
- 중복 태그 입력 시 사용자에게 즉각적인 피드백을 제공해야 한다.

### 범위 (Scope)

| 포함 | 제외 |
|------|------|
| 노트 편집 화면에서 태그 CRUD | 태그 기반 필터/검색 |
| 노트 목록 카드에 태그 배지 표시 | 태그 색상 사용자 지정 |
| 중복 방지 Toast 알림 | 태그 일괄 관리 화면 |
| 서버 영속성 (JSON Server PATCH) | 태그 간 관계/계층 구조 |

## 2. 사용자 스토리

### US-01: 태그 추가
> 노트를 작성하는 사용자로서, 태그를 추가하여 노트를 분류하고 싶다.

**수용 기준:**
- 태그 목록 끝에 `+` 버튼이 보인다.
- `+` 클릭 시 인라인 입력 필드가 뱃지 형태로 나타나고 포커스를 받는다.
- `Enter` 키 또는 블러(blur) 시 태그가 확정된다.
- `Esc` 키 입력 시 입력이 취소되고 필드가 닫힌다.
- 빈 문자열 입력 시 취소 처리된다(태그 추가 안 됨).

### US-02: 태그 편집
> 기존 태그를 수정하고 싶다.

**수용 기준:**
- 기존 태그 배지를 클릭하면 해당 배지가 인라인 입력 필드로 전환된다.
- 기존 태그 텍스트가 입력 필드에 미리 채워져 있다.
- `Enter` 또는 블러 시 변경 내용이 확정된다.
- `Esc` 또는 빈 문자열로 확정 시 취소된다(원래 값 유지).
- 수정 중인 태그와 중복되는 다른 태그 값으로 변경 시도 시 Toast 에러 표시.

### US-03: 태그 삭제
> 더 이상 필요 없는 태그를 삭제하고 싶다.

**수용 기준:**
- 각 태그 배지에 `×` 버튼이 있다.
- `×` 클릭 시 해당 태그가 즉시 제거된다.
- 클릭 이벤트가 부모 배지(편집 진입)로 전파되지 않는다.
- 편집 중인 태그의 `×`를 클릭하면 편집 상태도 함께 초기화된다.

### US-04: 중복 방지
> 이미 있는 태그를 실수로 다시 입력했을 때 알림을 받고 싶다.

**수용 기준:**
- 추가/편집 시 동일한 값의 태그가 이미 존재하면 Toast 에러가 표시된다.
- Toast 메시지: `"이미 존재하는 태그입니다."`
- 중복 감지 시 입력 필드는 닫히지 않고 유지된다(재입력 기회 제공).

### US-05: 노트 목록에서 태그 확인
> 노트 목록에서 각 노트에 붙은 태그를 미리 볼 수 있어야 한다.

**수용 기준:**
- 태그가 있는 노트 카드에는 태그 배지가 노트 내용 미리보기 아래에 표시된다.
- 태그가 없으면 배지 영역 자체가 렌더링되지 않는다.

## 3. 기능 명세

### 3.1 입력 제약

| 규칙 | 값 | 근거 |
|------|----|------|
| 최대 길이 | 10자 | 배지 레이아웃 깨짐 방지 (디자인 시스템 스펙) |
| 공백 처리 | trim() 후 검사 | 앞뒤 공백만 있는 태그 방지 |
| 중복 검사 | 대소문자 구분 일치 | `"React"` ≠ `"react"` |
| 최대 태그 수 | 제한 없음 (v1) | 추후 UX 검토 후 추가 가능 |

### 3.2 인터랙션 상태 머신

```
[idle]
  │ + 클릭
  ▼
[adding]  ──Enter/Blur──▶ 유효성 검사 ──중복──▶ [adding] + Toast
  │ Esc / 빈값                         ──통과──▶ [idle] (태그 추가됨)
  ▼
[idle]

[idle]
  │ 배지 클릭
  ▼
[editing(index)]  ──Enter/Blur──▶ 유효성 검사 ──중복──▶ [editing] + Toast
  │ Esc / 빈값                            ──통과──▶ [idle] (태그 변경됨)
  ▼
[idle]
```

**상태 변수:**

| 변수 | 타입 | 역할 |
|------|------|------|
| `isAdding` | `boolean` | 신규 태그 입력 필드 표시 여부 |
| `editingIndex` | `number \| null` | 편집 중인 태그의 인덱스 (`null` = 편집 없음) |
| `inputValue` | `string` | 현재 입력 필드 값 |

### 3.3 데이터 흐름

```
TagInput (로컬 상태)
  └─ onChange(tags[]) 호출
      └─ NoteEditor.tags (로컬 상태 업데이트)
          └─ handleSave() 시
              └─ NotesContext.addNote / editNote(tags 포함)
                  └─ api.createNote / updateNote (POST / PATCH)
                      └─ JSON Server db.json
```

- 태그 배열은 Note 객체의 `tags: string[]` 필드로 서버에 저장된다.
- 저장 전까지는 완전히 로컬 상태이며 서버에 즉시 반영되지 않는다.

### 3.4 Toast 알림 사양

| 상황 | 종류 | 메시지 | 지속 시간 |
|------|------|--------|-----------|
| 중복 태그 입력 | error | `이미 존재하는 태그입니다.` | 3000ms |

## 4. UI 명세

### 4.1 레이아웃 위치

```
NoteEditor
└── [본문 textarea]
└── ── 구분선 ──
└── TagInput          ← 태그 영역 (border-t 위)
└── ── 구분선 ──
└── [저장 / 취소 버튼]
```

### 4.2 컴포넌트 구조

```
TagInput
├── tag-badge[]       (기존 태그, 클릭 → 편집 모드)
│   ├── tag-text
│   └── × button
├── tag-badge[editing] (편집 중인 태그)
│   └── <input autoFocus>
├── tag-input-badge   (isAdding=true 일 때)
│   └── <input autoFocus placeholder="태그 입력">
└── + button          (isAdding=false 일 때)
```

### 4.3 시각 스타일 (현재 구현 기준)

현재 구현은 인라인 Tailwind 유틸리티를 사용한다. 향후 `Tag` 공용 컴포넌트의 `.tag .tag-{color}` 클래스로 마이그레이션 예정.

| 요소 | 현재 클래스 | 목표 클래스 |
|------|------------|------------|
| 태그 배지 | `px-2 py-0.5 rounded-full bg-foreground/10` | `.tag .tag-indigo` |
| 편집 중 배지 | `border border-foreground/30` 추가 | `.tag-input` |
| + 버튼 | `w-5 h-5 rounded-full bg-foreground/10` | 별도 정의 필요 |

## 5. 비기능 요구사항

| 항목 | 요구사항 |
|------|---------|
| 접근성 | `autoFocus`로 입력 필드 즉시 포커스 |
| 이벤트 격리 | 삭제 버튼 `e.stopPropagation()`으로 편집 진입 방지 |
| 반응성 | 태그 추가/삭제는 로컬 상태 업데이트로 즉각 반영 |
| 영속성 | 저장 버튼 클릭 시에만 서버 반영 (낙관적 업데이트 없음) |

## 6. 미구현 / 향후 개선 사항

| 항목 | 우선순위 | 비고 |
|------|---------|------|
| 최대 태그 수 제한 (예: 10개) | 낮음 | 현재 무제한 |
| 태그 최대 길이 10자 검증 | 중간 | 스펙에 명시되어 있으나 미구현 |
| 태그 색상 팔레트 자동 지정 | 낮음 | 공용 `Tag` 컴포넌트 마이그레이션 시 |
| 태그 기반 필터링 | 높음 | `FilterChip` 공용 컴포넌트 연동 |
| `TagInput` 공용 `Tag` 배지로 마이그레이션 | 중간 | 디자인 시스템 일관성 |
| 키보드 탐색 (Tab으로 배지 이동) | 낮음 | 접근성 개선 |
